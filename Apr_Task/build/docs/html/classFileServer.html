◆ create_socket()

void FileServer::create_socket ( )
inlineprivate

Создает и настраивает серверный сокет для принятия подключений

Выполняет последовательно:

  1. Получение информации о доступных адресах через getaddrinfo()
  2. Создание сокета для каждого возможного адреса
  3. Установка опции SO_REUSEADDR для быстрого переиспользования порта
  4. Привязка сокета к адресу (bind)
  5. Перевод сокета в режим прослушивания (listen)
Исключения
std::runtime_errorв случаях:
  • Ошибки получения адресов (getaddrinfo)
    • Неудачной привязки сокета (bind)
    • Ошибки перевода в режим прослушивания (listen)

inlinestaticprivate

Получает указатель на IP-адрес в структуре sockaddr.

Функция определяет тип адреса (IPv4/IPv6) и возвращает указатель на соответствующее поле с IP-адресом в унифицированном формате.

Аргументы
[in]saУказатель на структуру sockaddr (может быть sockaddr_in или sockaddr_in6)
Возвращает
Указатель на IP-адрес в виде:
  • struct in_addr* для IPv4 (AF_INET)
  • struct in6_addr* для IPv6 (AF_INET6)

◆ process_data()

static std::vector<int> FileServer::process_data ( std::vector< int > &  data)
inlinestaticprivate

Выполняется следующее:

  1. Передается вектор значений
  2. Проверяется, правильно ли передан вектор значений (является ли количество элементов в векторе плным квадратом)
  3. Из вектора значений пересобирается матрица смежности графа
  4. Матрица смежности передантся в функцию FindEulerPath, которая вычисляет эйлеров путь
  5. Возвращается вектор, хранящий вершины, по которым проходит эйлеров путь
  6. Бросает исключение, в случае ненахождения эйлерового пути
    Аргументы
    [in]dataВектор, хранящий в себе матрицу смежности
    Возвращает
    Возвращает эйлеров путь

◆ run()

void FileServer::run ( )
inline

Основной цикл работы сервера

Выполняет следующие действия в бесконечном цикле:

  1. Принимает входящие соединения через accept()
  2. Определяет IP-адрес клиента
  3. Создает дочерний процесс для обработки соединения
  4. В дочернем процессе вызывает обработчик соединения
    Исключения
    std::exceptionМожет пробросить исключения из:
  • inet_ntop()
  • handle_connection()
  • Системные ошибки

◆ setup_signal_handler()

void FileServer::setup_signal_handler ( )
inlineprivate

Настраивает обработчик сигнала SIGCHLD для автоматического завершения дочерних процессов.

Функция выполняет следующие действия: Создает обработчик сигнала, который вызывает waitpid() для всех завершенных дочерних процессов. Использует флаг SA_RESTART для автоматического перезапуска системных вызовов, прерванных сигналом. Устанавливает пустую маску сигналов для обработчика. Регистрирует обработчик для сигнала SIGCHLD.
